## Section 1: Physical Access & Mobility
Review the following physical requirements to determine the base controller type.

* **Hand Function:** Does the user have full use of both hands, or is there limited range of motion?
* **Strength/Fatigue:** Does the user experience muscle fatigue after short periods of play?
* **Grip:** Can the user hold a standard controller, or is a mounted solution required?

For more details, view our [Physical Assessment Guide](https://example.com/physical)¹.

## Section 2: Input Methods & Switches
Adaptive gaming often requires moving away from standard joysticks to specialized switches.

1.  **Button Size:** Are larger buttons (like Big Red switches) necessary for accuracy?
2.  **Mounting:** Does the equipment need to be attached to a wheelchair or desk?
3.  **Alternative Inputs:** Would the user benefit from a [Sip-and-Puff system](https://example.com/sip-puff)² or [Eye-Tracking software](https://example.com/eye-track)³?

Check our [Switch Compatibility Matrix](https://example.com/matrix)⁴ for hardware specs.

## Section 3: Software & In-Game Settings
Many modern games include built-in accessibility features that reduce the need for specialized hardware.

* **Visual Aids:** High contrast modes, subtitle scaling, and colorblind filters.
* **Motor Aids:** Toggle instead of hold, auto-run, and aim-assist levels.
* **Difficulty Scaling:** Can the game speed be slowed down?

Consult the [Game Accessibility Settings Database](https://example.com/settings)⁵ to see if your favorite game is supported.
